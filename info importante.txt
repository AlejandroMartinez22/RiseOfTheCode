Cada enemigo debe tener un script distinto por que tienen comportamientos diferentes, y la lógica cambia bastante.
Se crea un recurso con caracteristicas comunes de todos los enemigos, pero en aquellos que atacan a distancia el campo
de daño no se usa, ya que ese está definido por el proyectil que disparan.
Este recurso se llama enemydata.gd
Si queremos modificar alguna caracteristica como la velocidad o daño, se modifica el archivo .tres del enemigo, en la carpeta enemies

Layer -> Quien eres
Mask -> A quien puedes golpear

Layer 1 → Player
Layer 2 → Enemies
Layer 3 → PlayerBullets
Layer 4 → EnemyBullets
Layer 5 → Walls / Environment

Las balas si deben tener mask de los objetivos, pero los personajes no deben tener mask de las balas


Se creó un singleton para manejar especificamente el audio de muerte de los enemigos.
Esto se hace por que, si reproducimos el sonido de muerte dentro
del mismo script del enemigo, el audio se corta cuando el enemigo muere
y el nodo desaparece, por eso debemos reproducir el audio "en otra parte",
de manera que pueda concluir su reproducción incluso cuando el nodo del enemigo
se eliminó. Se opta por usar un singleton dado que habrán varios enemigos
en el juego, y es bastante probable que requiramos una lógica similar en caad uno.
